Forklar ganske kort hvordan du har løst opgaven
	Vi har taget DTD dokumentet og oversat hvert element og attribut til den respektive element og attribute i XML schema. Derefter har vi tilføjet de rigtige typer (det der stod som ikke muligt i DTD). Endelig kikkede vi på de krav der overhovedet ikke kunne opfyldes i DTD (18 og 19).

Hvilke udfordringer havde du her?
	Der skulle skrives regular expressions til at forhindre begyndende whitespaces m.m.  

Hvilke krav kan ikke valideres med XML Schema?
	Requirement 18: Due to requirement that a node validation must be self-contained we cannot make requirements about a node's siblings. In Schematron this is possible with <sch:assert test="not(following-sibling::text)">
 for text and something similar for ws.
	Requirement 19: Same problem as 18.
	The only way to solve this is to write all legal combinations of sequences, which is possible but rather verbose and not easy to maintain/modify.
